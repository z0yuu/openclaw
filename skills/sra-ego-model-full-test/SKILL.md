# Model Full Test (模型的完整测试)

## Skill Scope

This skill handles real full model-test execution with strict serial stages: local compile first, then local run.

## Routing Boundary

- If user intent is full serial compile+run (`model-test`), route to this skill.
- If user explicitly asks only compile or only run, route to the corresponding single-stage skill.

## Entry Gate

- `USER_ID_OPENAPI` is required by index gate before task start.
- In each stage, required fields still follow child skill contracts.
- For portal mode in each stage, use current environment variable `USER_ID_OPENAPI` by default; only ask user to set env when missing.

## Boundary

This skill orchestrates stage sequencing and gates. Detailed compile/run execution contracts are delegated to child skills.

## Trigger And Interaction Policy

Use this skill when the user wants a serial local compile plus local run workflow, for example:

- "run local compile and then local run"
- "help me run local compile + local run in sequence"
- "compile first, then start local run after compile finishes"

When triggered:

1. Launch `compile_agent`.
2. `compile_agent` collects compile required inputs and blocks until inputs are complete.
3. `compile_agent` executes compile and returns outputs.
4. Launch `run_agent` only after compile success.
5. `run_agent` collects run required inputs and blocks until inputs are complete.
6. `run_agent` executes local run and returns final status/logs.

Guardrails:

1. Do not merge compile and run into one execution context.
2. Use a fresh new agent for each stage (`compile_agent`, `run_agent`).
3. Parent/orchestrator must not fabricate required inputs; child stages should first extract from user natural language, then ask only for missing fields.
4. Parent/orchestrator and child agents must not use repo example files as default inputs.
5. Child agents must follow their own skill contracts:

- compile stage: `../sra-ego-model-local-compile/SKILL.md`
- run stage: `../sra-ego-model-local-run/SKILL.md`

6. Do not start compile execution before compile inputs are complete.
7. Do not start run stage before compile stage succeeds.
8. Do not start run execution before run inputs are complete.
9. If compile stage fails, stop the whole workflow and return compile result + explicit failure reason.
10. Every required input must come from explicit user-provided values in current conversation (or explicit user-approved reuse from prior turn).
11. Before each stage execution, echo resolved inputs and block until user confirms execution.
12. In sra-ego-model-full-test workflow, `output_path` is generated by agent, not provided by user.
13. When user asks to execute real `model-test`, do not run any unit-test workflow first.
14. Do not use any `unit-tests` directory input as compile/run inputs.
15. `unit-tests` scripts are only allowed when user intent explicitly includes "unit test", "单测", or "run_cases.sh".
16. Natural-language examples such as `compile model.py` or `run source_job_id=xxx with a given image` should be parsed by child skills (`sra-ego-model-local-compile` / `sra-ego-model-local-run`) before asking follow-up fields.

## Step 1: Launch `compile_agent`

Start a new agent bound to:

- skill: `../sra-ego-model-local-compile/SKILL.md`
- role: collect compile inputs, validate, execute compile, return artifacts

Compile agent must:

1. Ask user for all required compile fields.
2. Wait for user input when any required field is missing.
3. Reject invalid fields per compile skill constraints.
4. Do not substitute missing fields with any defaults from `example/` or previous local trials.
5. Generate a temporary `output_path` for compile artifacts under the system temp area.
6. Echo resolved inputs (including generated `output_path`) before execution and wait for explicit user confirmation.
7. If user explicitly provides `output_path`, use user value; otherwise use agent-generated temp path.
8. Apply compile skill extraction-first policy: parse `entry_file/source_dir/compile_image` from user text first, then ask for missing required fields.

### Step 1 Input Display Contract (must follow)

To avoid confusion, compile input collection must be shown as "two code-file source types + one common section".

Display template:

```text
Please provide the Stage 1 compile inputs:
Choose exactly one source group: `code files source A` or `code files source B`.
[code files source A: local]
- source_dir
- entry_file

[code files source B: portal]
- model_id
- model_version_id
- cluster (sg/us/uat)
- USER_ID_OPENAPI (default: env USER_ID_OPENAPI; optional override)

[common]
- compile_image
- EGO_MODE (TRAINING or SERVING)
```

`output_path` is not a user input in this workflow. Agent must auto-generate it under the system temp area, for example:

```text
<system-temp>/sra-ego-model-full-test/compile/<timestamp>-<rand>
```

Rules:

1. Explicitly state: user only needs to provide one source type (`A` or `B`), not both.
2. Keep source-specific fields grouped under each source type.
3. Always show common fields separately from source-specific fields.
4. Do not ask user to provide `output_path`.

Required compile outputs returned to orchestrator:

- `compile_status` (`success` or `failed`)
- `material_dir`
- `entry_file`
- `compile_output_path`
- `compile_output_link` (path link shown to user)
- `compile_image`
- `EGO_MODE`
- `failure_reason` (when failed)

## Step 2: Compile Success Gate

Before launching `run_agent`, all conditions must pass:

1. `compile_status=success`
2. `compile_output_path` exists
3. Artifacts exist in `compile_output_path`:

- `ego-config.json`
- `ego-config.pb`
- `graph.pb`
- `graph.readable`

If any check fails, stop and return compile-stage failure diagnostics.
On stop, still return `compile_output_path` and `compile_output_link` for inspection.

Failure handling rule (must follow):

1. If Step 1 compile fails, terminate full workflow immediately.
2. Do not launch `run_agent`.
3. Explicitly return:

- `compile_stage.status=failed`
- `compile_stage.failure_reason`
- failed command and exit code (if available)
- `compile_output_path` and `compile_output_link` (if created)

## Step 3: Launch `run_agent`

Start a new agent bound to:

- skill: `../sra-ego-model-local-run/SKILL.md`
- role: collect run inputs, validate, execute local run, report logs

Input handoff rule:

- `model_dir = compile_output_path`

`model_dir` source rule (must follow):

1. If user explicitly provided compile `output_path`, then `model_dir` must use that exact path.
2. If user omitted compile `output_path`, compile skill generates temp path; `model_dir` must use that generated path.
3. Do not ask user to re-enter `model_dir` in run stage when it is already derived from compile output.

Run stage then collects one mode of required inputs:

Mode A (local explicit):

- `converter_dir` (required)
- `learner_config` (required)
- `runtime_image` (required)
- `log_dir` (required)
- `run_config` (optional)

Mode B (portal source):

- `source_job_id` (required)
- `cluster` (required, `sg/us/uat`)
- `USER_ID_OPENAPI` (default: env USER_ID_OPENAPI; optional override)
- `runtime_image` (required)
- `log_dir` (required)
- `run_config` (optional override)

Run agent must:

1. Wait for user input when required fields are missing.
2. Validate required files in `model_dir` (`ego-config.json`, `ego-config.pb`).
3. Do not substitute missing fields with any defaults from `example/` or previous local trials.
4. Execute only after selected mode required inputs are complete and user explicitly confirms:
5. Apply run skill extraction-first policy: parse `source_job_id` / `converter_dir` / `learner_config` / `runtime_image` from user text first, then ask for missing required fields.

```bash
sh ../sra-ego-model-local-run/scripts/host_launch_local_job.sh \
  --model-dir "<model_dir>" \
  --runtime-image "<runtime_image>" \
  --log-dir "<log_dir>" \
  [--converter-dir "<converter_dir>" --learner-config "<learner_config>"] \
  [--source-job-id "<source_job_id>" --cluster "<cluster>" --user-id-openapi "<USER_ID_OPENAPI>"] \
  [--run-config "<run_config>"]
```

Run execution always uses the run skill launcher to re-assemble a temporary `job_dir` from split inputs (`model_dir`, `converter_dir`, `learner_config`, optional `run_config`, fixed `flags.txt`) before container startup.

### Step 3 Input Display Contract (must follow)

Run input collection must also be shown in two phases.

Phase A: ask run mode only

```text
Please choose the run input mode first:
1) local
2) portal
```

Phase B: show only selected mode fields + common fields

If user selects `local`, display:

```text
You selected `local`. Please provide these required fields:
[local mode]
- converter_dir
- learner_config

[common]
- runtime_image
- log_dir
```

If user selects `portal`, display:

```text
You selected `portal`. Please provide these required fields:
[portal mode]
- source_job_id
- cluster (sg/us/uat)
- USER_ID_OPENAPI (default: env USER_ID_OPENAPI; optional override)

[common]
- runtime_image
- log_dir
```

Optional for both modes:

- run_config

Rules:

1. Do not present `local` and `portal` required fields in the same checklist.
2. Explicitly label mode-specific fields vs common fields.
3. If run mode is not chosen, only ask for run mode; do not ask other fields yet.
4. In portal mode, prefer current environment variable `USER_ID_OPENAPI`; only ask user to set env when it is missing.

## Step 4: Final Output Contract

Always return a structured stage summary:

1. `compile_stage`

- `status`
- `compile_output_path`
- `compile_output_link`
- `material_dir`
- `entry_file`

2. `run_stage`

- `status`
- `model_dir`
- `runtime_image`
- `log_dir`
- `input_mode` (`local` or `portal`)
- `converter_dir`/`learner_config` or `source_job_id`/`cluster`

3. `logs`

- `<log_dir>/coordinator.log`
- `<log_dir>/worker.log`
- `<log_dir>/sample_server.log`

4. `resource_summary`

- docker resource summary fields from `host_launch_local_job.sh`

## Failure Output Contract

On any failure, include:

- `failed_stage` (`compile` or `run`)
- failed command
- exit code
- explicit missing input/file diagnostics
- `compile_output_path` (if already created)
- `compile_output_link` (if already created)
